从 Claude Code 迁移到 Codex CLI / Codex App

这是一份给已有 Claude Code 使用习惯的开发者看的 Codex 上手手册：不从零解释“AI 编程助手”，只讲你每天会按什么、敲什么、怎么控制权限、怎么把 Claude 的肌肉记忆迁移过来。

$ codex --full-auto > 找出登录页测试失败原因，并做最小修复 Codex 会读代码、提出计划、编辑文件、 运行命令，并在必要时请你确认。

1. 心智模型

Codex CLI

终端里的本地编码代理。运行在当前目录，能读写文件、执行命令、做 code review，也能非交互式执行任务。

Codex App

桌面版控制中心。适合多项目、多线程、并行代理、内置 worktree、Git、终端、浏览器、自动化和可视化审阅。

共享概念

两者都围绕工作区、权限、上下文、工具、规则文件和可复用技能。CLI 更像键盘流，App 更像任务驾驶舱。

迁移建议：如果你习惯 Claude Code 的 REPL，就先用 codex；如果你想同时跑多个任务、看 diff、开本地网页或管理工作树，就用 Codex App。

2. 快速开始

安装与登录

# npm 安装或升级
npm i -g @openai/codex@latest

# 启动交互式 TUI
codex

# 查看登录状态
codex login status

# 登录：默认打开浏览器进行 ChatGPT OAuth
codex login

# 用 API key 登录
printenv OPENAI_API_KEY | codex login --with-api-key

第一次推荐这样跑

# 在项目目录启动，先让 Codex 解释项目
codex "请先阅读这个项目，说明目录结构、启动方式、测试方式，不要改文件"

# 低摩擦自动模式：允许 workspace-write，并在 Codex 判断需要时询问
codex --full-auto "修复当前测试失败，保持改动最小"

# 非交互式任务，适合脚本或 CI
codex exec --cd /path/to/repo "运行测试并总结失败原因"

Codex App 启动

# 从 CLI 打开桌面 App
codex app

# macOS 上可打开指定工作区
codex app /path/to/repo

3. Claude Code 到 Codex 的命令映射

Claude Code 习惯 Codex 对应 说明
claude codex 启动交互式终端会话。
claude "query" codex "query" 启动时带上首条提示。
claude -p "query" codex exec "query" 或 codex e "query" 非交互式执行并退出。Codex 支持 --json、-o、--output-schema 等自动化参数。
claude -c / --continue codex resume --last 继续最近的交互式会话。
claude --resume <id> codex resume <session-id> 按会话 ID 恢复。
claude update npm i -g @openai/codex@latest npm 安装的 Codex 用 npm 升级；也可以查看你的安装渠道是否有其他升级方式。
claude mcp codex mcp 管理 MCP 服务器。
CLAUDE.md AGENTS.md 项目长期指令文件。Codex 会读取相关目录中的 AGENTS.md。
.claude/settings.json ~/.codex/config.toml Codex 以 TOML 配置为主，命令行 -c key=value 可临时覆盖。

4. CLI 命令总览

以下命令来自本机 codex --help 和 OpenAI 官方 CLI reference。你的本机版本为 codex-cli 0.118.0。

命令 用途 常见用法
codex启动交互式 TUI。codex --full-auto
codex exec / codex e非交互式执行任务。codex e --json "检查测试"
codex review非交互式代码审查。codex review
codex login / logout管理认证。codex login status
codex mcp管理 MCP 服务器。codex mcp list
codex app启动 Codex 桌面 App。codex app /path/to/repo
codex completion生成 shell completion。codex completion zsh
codex sandbox在 Codex 沙盒中运行命令。高级/调试场景。
codex apply / codex a应用 Codex Cloud 任务产生的最新 diff。codex apply
codex resume恢复旧会话。codex resume --last
codex fork从旧会话分叉新线程。codex fork --last
codex cloud浏览或执行 Codex Cloud 任务。实验功能。
codex features查看/切换 feature flags。codex features list

常用全局参数

参数作用建议
-C, --cd <DIR>指定工作目录。脚本里强烈建议显式写。
-m, --model <MODEL>指定模型。临时切模型用；长期默认写配置。
-s, --sandboxread-only、workspace-write、danger-full-access。日常用 workspace-write。
-a, --ask-for-approvaluntrusted、on-request、never。交互用 on-request，自动化用 never。
--full-auto低摩擦预设：on-request + workspace-write。最适合从 Claude Code 迁移后的日常开发。
-i, --image <FILE>附加图片输入。给截图、设计稿、报错截图时用。
--search启用实时 web search。查最新文档、版本、法规时用。
-c, --config key=value临时覆盖配置。适合一次性调整模型、profile、features。
--dangerously-bypass-approvals-and-sandbox跳过确认和沙盒。只在外部已强隔离环境用。

5. CLI Slash 命令

在交互式 codex 里输入 / 会出现命令列表。任务运行中也可以输入 slash 命令并按 Tab 排队到下一轮。

命令用途Claude Code 用户理解方式
/permissions调整当前会话权限。类似切换允许/询问策略。
/model切模型和推理强度。对应 Claude 的 /model。
/fast切换 Fast mode。需要速度优先时用。
/plan进入计划模式。让 Codex 先规划再动手。
/personality切换回复风格。控制表达方式，不等于改系统能力。
/status查看模型、权限、上下文等。对应 Claude 的状态/用量检查习惯。
/compact压缩上下文。长会话续航必备。
/clear清屏并开始新聊天。想重开上下文时用。
/diff查看 Git diff。审阅 Codex 改了什么。
/review审查当前工作区。提交前让另一个视角挑问题。
/init生成 AGENTS.md 脚手架。对应 Claude 的 /init 生成 CLAUDE.md。
/mcp查看 MCP 工具。确认外部工具是否连上。
/mention把文件或文件夹附到对话。类似 Claude 的 @file 指向上下文。
/apps浏览并插入 app connector。App/连接器能力入口。
/plugins管理插件。扩展 Codex 能力。
/agent切换子代理线程。并行任务或探索线程时用。
/ps查看后台终端输出。长命令跑着时检查状态。
/stop停止后台终端。取消当前会话启动的后台工作。
/fork分叉当前对话。保留主线，试另一个方案。
/resume恢复旧会话。继续历史任务。
/new同一 CLI 内开新对话。不退出但重开任务。
/copy复制最新输出。也可按 Ctrl + O。
/quit / /exit退出 CLI。退出前确认重要改动已保存。

6. Codex App

Codex App 是桌面端的“任务中心”：项目侧边栏、线程、review 面板、终端、浏览器、worktree、自动化都集中在一个窗口里。官方文档说明，App 可在 macOS 和 Windows 使用。

适合 App 的场景

  • 一个项目里同时跑多个线程。
  • 用内置 worktree 隔离并行改动。
  • 在同一界面看 diff、stage、commit、push。
  • 打开本地网页，让 Codex 查看或操作浏览器。
  • 创建自动化，让线程按计划醒来检查事情。

App 快捷键

  • 命令菜单：Cmd + Shift + P 或 Cmd + K
  • 设置：Cmd + ,
  • 打开文件夹：Cmd + O
  • 新线程：Cmd + N
  • 查找线程：Cmd + F
  • 切换侧边栏：Cmd + B
  • 切换终端：Cmd + J

App slash 命令

命令用途
/feedback打开反馈对话框，可附带日志。
/mcp打开 MCP 状态，查看连接的服务器。
/plan-mode切换多步骤规划模式。
/review进入代码审查模式，可审查未提交改动或与 base branch 比较。
/status显示线程 ID、上下文使用量和速率限制。

Deeplink

codex://settings
codex://skills
codex://automations
codex://threads/<thread-id>
codex://new?prompt=总结这个项目&path=/absolute/path/to/repo

7. 权限与沙盒

Claude Code 用户迁移时最重要的差异之一，是 Codex 把“是否能执行”和“能访问哪里”拆得很清楚。

维度选项含义
Approval policy untrusted、on-request、never 决定什么时候需要人工确认。
Sandbox read-only、workspace-write、danger-full-access 决定文件系统写权限边界。
快捷预设 --full-auto 等价于低摩擦开发常用组合：工作区可写，并在需要时询问。
谨慎使用：--dangerously-bypass-approvals-and-sandbox 会跳过确认和沙盒，只适合已经由外部容器、虚拟机或 CI runner 强隔离的环境。 
# 只读探索
codex -s read-only "解释这个仓库的构建流程"

# 日常开发
codex --full-auto "实现登录页错误提示，并运行相关测试"

# 自动化脚本，失败时不要停下来问我
codex exec -a never -s workspace-write "格式化并运行单元测试"

8. 配置、规则、记忆

~/.codex/config.toml

Codex 的默认配置主要在 ~/.codex/config.toml。命令行里的 -c key=value 会覆盖本次运行的配置。

# 临时切模型
codex -c model='"gpt-5.4"' "解释这段代码"

# 使用 profile
codex --profile work

# 查看 feature flag
codex features list
codex features enable <feature>
codex features disable <feature>

AGENTS.md

把 Claude Code 里的 CLAUDE.md 思路迁到 Codex 时，通常写成 AGENTS.md：项目结构、命令、测试、代码风格、禁止事项、PR 规范都放这里。

# 在 Codex CLI 里生成脚手架
/init

建议写进 AGENTS.md 的内容

  • 项目启动命令、测试命令、lint 命令。
  • 哪些目录不能随便改，例如 generated、vendor、migration 历史。
  • 提交前必须跑哪些验证。
  • 代码风格：命名、错误处理、日志、国际化、UI 规范。
  • 安全红线：不要打印密钥、不要访问生产数据、不要执行破坏性命令。

9. MCP、Skills、Plugins

MCP

把外部工具或数据源接给 Codex。常用命令包括 codex mcp list、codex mcp add、codex mcp remove、codex mcp login。

Skills

可复用的工作流说明。App 中输入 $ 可以显式调用技能；启用的 skills 也可能出现在 slash 命令列表里。

Plugins

插件可以打包 skills、MCP server、工具和应用集成。适合把团队内部工具变成 Codex 可发现能力。

# MCP 示例
codex mcp list
codex mcp add docs --url https://example.com/mcp
codex mcp get docs --json
codex mcp remove docs

10. 常用工作流

探索codex -s read-only
先让 Codex 读项目、画边界。
计划用 /plan 或明确说“先给计划”。
实施--full-auto 适合日常小中型改动。
验证让 Codex 跑测试、lint、类型检查。
审阅/diff、/review、App review 面板。

修 bug

codex --full-auto "复现并修复这个 bug：粘贴报错。先找根因，再做最小改动，最后运行相关测试"

理解陌生项目

codex -s read-only "请阅读项目，不要改文件。输出：架构概览、关键入口、运行命令、测试命令、风险点"

做代码审查

codex review

# 或在交互式会话中
/review

用图片作为输入

codex -i screenshot.png "根据截图修复页面布局问题，注意移动端不要重叠"

长会话续航

/status
/compact 请保留当前任务目标、已改文件、失败测试、下一步计划

11. 提示词模板

Claude Code 老用户最顺手的写法

请先阅读相关文件，确认根因后再修改。
要求：
1. 改动尽量小。
2. 不要重构无关代码。
3. 修改前说明你要动哪些文件。
4. 修改后运行最相关的测试。
5. 最后总结改动和验证结果。

让 Codex 少走弯路

这个任务的成功标准是：
- 用户可以完成 X。
- Y 情况下不能回归。
- 必须通过 npm test -- login.spec.ts。

约束：
- 不要改数据库 schema。
- 不要引入新依赖。
- 保持现有 UI 风格。

需要 App 多线程时

请把这个任务拆成可以并行的子任务。
每个子任务说明：
- 目标
- 负责文件范围
- 验证命令
- 是否需要 worktree 隔离
