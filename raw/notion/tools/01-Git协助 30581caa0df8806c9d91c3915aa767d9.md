# 📌 模块一：Git 核心概念（够用版）

## 1️⃣ Repository（仓库）

项目的版本管理空间

- 本地仓库：你电脑
- 远程仓库：GitHub 上

---

## 2️⃣ Commit（提交）

一次版本记录。

```
git add .
git commit -m "说明"
```

作用：

- 记录改动
- 可回滚
- 可对比历史

---

## 3️⃣ Branch（分支）

独立开发线路。

常见：

- main（主分支）
- feature/xxx（功能）
- fix/xxx（修复）

意义：

- 防止污染主分支
- 多功能并行开发

---

## 4️⃣ Pull Request（PR）

请求把分支合并到主分支。

流程：

1. 开发完成
2. push
3. 创建 PR
4. review
5. merge

即使一个人开发也建议用 PR。

---

## 5️⃣ Merge（合并）

把一个分支合并进另一个分支。

---

## 6️⃣ Remote / origin

远程仓库地址。

```
git remote -v
```

默认远程叫 origin。

---

## 7️⃣ GitHub Actions

自动化流程（CI/CD）：

- 自动测试
- 自动构建
- 自动部署

文件位置：

```
.github/workflows/*.yml
```

---

# 📌 模块二：Git 常用命令（高频实用）

## 🔹 基础查看

```
git status        # 当前状态
git log           # 查看历史
git diff          # 查看改动
```

---

## 🔹 分支操作

```
git branch                    # 查看分支
git branch feature/login      # 创建分支
git checkout feature/login    # 切换分支
git checkout -b feature/test  # 创建并切换
git branch -d feature/login   # 删除分支
```

（新版推荐）

```
git switch -c feature/test
git switch main
```

---

## 🔹 提交流程

```
git add .
git commit -m "feat: 新增登录功能"
```

---

## 🔹 远程操作

```
git pull            # 拉取远程更新
git push            # 推送
git push -u origin feature/login
```

---

## 🔹 合并

```
git merge feature/login
```

---

## 🔹 克隆仓库

```
git clone 仓库地址
```

---

# 📌 模块三：常见 Git 专业术语

| 名词 | 含义 |
| --- | --- |
| Repo | 仓库 |
| Commit | 一次提交记录 |
| Branch | 分支 |
| PR | Pull Request |
| Merge | 合并 |
| Rebase | 线性重放历史 |
| Remote | 远程仓库 |
| origin | 默认远程名 |
| Fork | 复制仓库 |
| CI/CD | 自动测试部署 |

# 📌 模块四：Git 推荐工作流（个人开发）

推荐结构：

```
main
  ↓
feature/xxx
  ↓
开发
  ↓
创建 PR
  ↓
merge
```

即使一个人开发也建议：

- 不直接改 main
- 用 feature 分支
- 用 PR 合并

好处：

- 清晰历史
- 安全
- 可回滚
